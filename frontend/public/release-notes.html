Release Notes

A running log of updates, fixes, and improvements to Portfolio Monitor.

May 2026

May 19, 2026 Feature

Add automated daily IV ingestion worker

A new background worker (historical-iv-worker.js) automatically fetches and stores ATM implied volatility for major tickers (SPY, QQQ, IWM, AAPL, MSFT, and more) every weekday at 5 PM EST — one hour after market close. The worker logs each ticker's success or failure and can be run standalone with node scripts/historical-iv-worker.js. This ensures the IV history database is populated daily without manual intervention.

May 19, 2026 Feature

Add historical IV tracking for IVR/IVP calculations

The Options Scanner now stores daily implied volatility data per ticker, enabling accurate IV Rank (IVR) and IV Percentile (IVP) calculations based on a ticker's own volatility history rather than the current chain-wide IV distribution. IVR/IVP values fall back to chain-wide distribution when insufficient historical data is available.

May 18, 2026 Fix

Fix option chain calls always parsing as puts in the backend

The Schwab API returns option type in a putCall field ("CALL"/"PUT"), not optionType ("C"/"P"). The backend was checking a non-existent field, causing all options — including calls from callExpDateMap — to be classified as puts. Also fixed the volume field to read totalVolume instead of a non-existent vol field. Both sides of the option chain now render correctly.

May 18, 2026 Fix

Fix Calls column in Option Chain showing no data

The Calls side of the side-by-side Option Chain was rendering empty even though Calls existed in the ±10% moneyness range. Fixed a React Rules of Hooks violation where useMemo was called after an early return, causing inconsistent hook state. Both Calls and Puts now display correctly side-by-side.

May 18, 2026 Improvement

Redesign Option Chain layout and filtering

The Option Chain has been completely redesigned for better readability and focus. It now filters out deep OTM options (showing only strikes within ±10% of the underlying price) and displays Calls and Puts side-by-side. The At-The-Money (ATM) strike is visually anchored in the center with a subtle glow, and the table density has been increased to show more relevant data at a glance.

May 18, 2026 Improvement

Add liquidity and moneyness filters to Best Opportunities ranking

Deep OTM options with negligible volume and open interest no longer clutter the Best Opportunities list. Options must now have at least some trading volume or 10+ open interest, and their strike price must be within 15% of the underlying price to qualify. This ensures ranked opportunities are actually tradeable.

May 18, 2026 Improvement

Refine Options Scanner UI with top 5 highlighting

Reduced the Best Opportunities table to show the top 5 contracts instead of 20 for better focus. The top 5 opportunities are now individually color-coded (Gold, Cyan, Magenta, Orange, Lime) across both the table and the 3D volatility surface for instant visual identification. Added a brief flash animation to the option chain when selecting a contract from the 3D chart to provide clear visual feedback.

May 18, 2026 Fix

Exclude 0DTE options from Best Opportunities ranking

Options expiring today (0DTE, DTE < 1) are now excluded from the global Best Opportunities table. Their near-zero vega, extreme IV smile, and tick-size noise produce unreliable edge estimates that drown out longer-dated opportunities. Select a specific expiration from the dropdown to view its chain directly.

May 18, 2026 Improvement

Enhance Options Scanner with Global DTE Filter and UI Fixes

Added a prominent global Date to Expiration (DTE) filter that updates the 3D volatility surface, best opportunities table, and option chain simultaneously. Fixed 3D plot color coding to clearly distinguish underpriced (green) and overpriced (red) options against a refined glass-morphism surface. Added a detailed interpretation guide for the Gamma Exposure (GEX) profile.

May 18, 2026 Improvement

Add interactive option chain highlighting from 3D surface

Clicking any scatter point on the 3D volatility surface now automatically selects the corresponding expiration date and highlights the specific option contract in the data table below, making it easier to drill down into specific opportunities.

May 18, 2026 Improvement

Enhance 3D volatility surface visualization

Improved the Options Scanner's 3D volatility surface chart by color-coding scatter points based on their theoretical edge. Green points indicate underpriced options, red points indicate overpriced options, and white points are fairly priced. Added detailed hover tooltips and a clear visual guide below the chart.

May 18, 2026 Feature

Add Options Scanner UI with 3D volatility surface

Introduced a new Options Scanner tab on the stock detail page. Features an interactive 3D volatility surface chart (SVI fitted vs market IV), a ranked list of the best spread-adjusted edge opportunities, full option chains with inline edge highlighting, and a Gamma Exposure (GEX) profile sidebar.

May 18, 2026 Feature

Add options scanner API with SVI surface fitting

New GET /api/options/scan/:ticker endpoint provides a comprehensive options scan: realized volatility, IV Rank/Percentile, Gamma Exposure (GEX), SVI volatility surface fitting per expiration, and spread-adjusted edge for each contract. Powers advanced quantitative options analysis.

May 16, 2026 Fix

Fix missing EV/EBITDA and ratio fields in portfolio dashboard

Resolved enterpriseToEbitda, pegRatio, sector, and industry fields always showing null by introducing a three-phase resolution: fast batch quote, summary cache lookup, and parallel quoteSummary calls for uncached tickers.

May 16, 2026 Improvement

Faster earnings profile loading

Parallelized API calls and added aggressive caching to the earnings profile endpoint, significantly reducing load times.

May 16, 2026 Improvement

Faster insider trading data

Optimized SEC API requests and added module-level caching, reducing load times for insider trading data by up to 80%.

May 16, 2026 Improvement

Faster portfolio loading

Replaced sequential API calls with batch fetching, reducing portfolio load times by up to 10x.

May 16, 2026 Improvement

Optimize memory usage

Added bounds to in-memory caches to prevent memory leaks and out-of-memory crashes under high load.

May 16, 2026 Security

Rate-limit DCF endpoint and cap simulation count

The DCF valuation endpoint now enforces a maximum of 5 requests per minute per IP address, and the simulations query parameter is capped at 5,000 (down from 100,000). This prevents resource exhaustion from abusive or runaway Monte Carlo simulation requests.

May 16, 2026 Security

Generate strong random password for default admin account on first setup

The default admin account (admin@stock-dashboard.local) now receives a cryptographically strong random password on first creation instead of using a hardcoded default. The generated credentials are printed clearly to the console at startup. Existing admin accounts are unaffected — their password is never changed.

May 16, 2026 Security

Remove real API keys from .env and add backend credentials to .gitignore

Replaced all hardcoded API keys (FRED, Schwab) in backend/.env with placeholders, and generated a strong 64-character random JWT secret. Added backend/.env, backend/.schwab-token.json, and backend/certs/*.pem to .gitignore to prevent accidental credential commits. Created backend/.env.example showing required vars without exposing secrets.

May 16, 2026 Fix

Fix server connection and login errors caused by restart loop

Resolved a critical backend issue where automated database and file updates triggered an infinite restart loop. Users can now reliably log in and use the application without encountering unexpected end-of-JSON errors.

May 16, 2026 Fix

Fix five backend data regressions

Patched missing try/finally in SEC ticker loading so a failed fetch doesn't permanently block future requests. Added missing await in live price refresh so stale tickers actually get updated before the response returns. Added optional chaining for financial statement and earnings history data to prevent crashes on incomplete API responses. Portfolio summaries now correctly map back to the requested tickers, returning an error entry for any missing quote instead of silently dropping it. Moved the earnings profile cache into the shared cache module so it is included in cache stats and flush operations.

May 16, 2026 Security

Fix reflected XSS in OAuth callback server

Error parameters from the Schwab authorization callback are now HTML-escaped before being rendered in the response page, preventing injection of malicious scripts via crafted redirect URLs.

May 15, 2026 Improvement

Simplify metric cards by removing sparklines

The metric cards in the Fundamentals tab have been simplified by removing the SVG sparkline charts, leaving a cleaner design that focuses on the YoY trend badge. Additionally, historical data has been wired up for more metrics.

May 15, 2026 Improvement

Update key metrics to show historical trend sparklines

The Fundamentals tab now displays historical trend sparklines and YoY percentage changes for key metrics instead of peer comparisons, providing better insight into a company's performance over time.

May 14, 2026 Fix

Fix live price 429 rate limit errors during market hours

Live price updates were hitting the global rate limiter (100 req/min shared across all endpoints) because the /portfolio/live endpoint had no dedicated rate limit. Added a separate limiter (30 req/min) for live prices, fixed the useLivePrices hook to prevent duplicate polling intervals when tickers change, and added explicit 429 detection with immediate 120s backoff. Polling now stops gracefully after 5 consecutive failures instead of hammering the server.

May 14, 2026 Feature

Add automatic default admin user on startup

The backend now auto-creates a default admin account (admin@stock-dashboard.local / ChangeMe123!) when it starts and no admin user exists yet. No more manual promote-user step needed for the first admin setup — admins should change the password after first login.

May 14, 2026 Fix

Fix Admin Dashboard cache stats displaying as dashes

Cache statistics in the Admin Dashboard now show actual values instead of “—”. The backend was returning per-segment cache stats that the frontend couldn't read — stats are now aggregated across all cache layers so keys, hits, misses, and sizes display correctly.

May 14, 2026 Feature

Add Admin Dashboard UI with cache management and margin debt tools

Admin users now have a dedicated dashboard visible in the navigation header. The dashboard displays Redis cache statistics (keys, hits, misses, key size, value size) and provides one-click buttons to flush the cache or trigger a manual margin debt update. Non-admin users who navigate to ?page=admin see an access denied message.

May 14, 2026 Feature

Add Admin role with protected management routes

Introduces role-based access control with an ADMIN role. Cache management and margin debt update endpoints now require admin privileges instead of localhost origin. Use node scripts/promote-user.js <email> to promote a user to admin.

May 14, 2026 Fix

Fix admin routes always returning 403 Forbidden

Corrected a middleware ordering bug where the admin role check ran before authentication, causing all admin-protected endpoints to reject even valid admin users. The authentication middleware now runs first so the user identity is available when the role check executes.

May 14, 2026 Feature

Add Sentry error tracking for frontend and backend

Errors are now automatically reported to Sentry from both the browser and server. Frontend errors include session replays for debugging user interactions. Backend errors include performance profiling. Set SENTRY_DSN and VITE_SENTRY_DSN environment variables to enable — Sentry is safely disabled when no DSN is configured.

May 14, 2026 Feature

Add user accounts with persisted portfolio and wishlist

Portfolio and wishlist data is now stored in the database instead of localStorage. New users can create an account or sign in to access their saved tickers from any device. The dashboard requires authentication — unauthenticated visitors see a login screen. A logout button is available in the header.

May 14, 2026 Improvement

All API endpoints now validate request data using Zod schemas, returning clear error messages for invalid email, password, ticker, or query parameters. A centralized error handler catches all errors and returns a consistent JSON response format. Prisma database errors are also caught and sanitized — no internal details leak in production.
